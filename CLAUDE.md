# CLAUDE.md

treemapify is an R package providing ggplot2 geoms for drawing treemaps, published on CRAN.

## Package structure

- `R/` — package source: geoms (`geom_treemap.R`, `geom_treemap_text.R`, `geom_treemap_subgroup_border.R`, `geom_treemap_subgroup_text.R`), layout algorithms (`treemap_squarified.R`, `treemap_fixed.R`), the legacy `treemapify()` function, data, and utilities
- `tests/` — testthat tests and vdiffr visual regression snapshots (in `tests/testthat/_snaps/`)
- `vignettes/` — package vignette (built with knitr/rmarkdown)
- `man/` — roxygen2-generated documentation (do not edit by hand)
- `docs/` — pkgdown site

## Conventions

- Standard CRAN R package layout
- Roxygen2 for documentation with markdown enabled (`Roxygen: list(markdown = TRUE)` in DESCRIPTION)
- testthat for unit tests; vdiffr for visual regression tests
- Australian English throughout (`Language: en-GB` in DESCRIPTION) — use "colour" not "color", "centre" not "center", etc.
- CI runs R CMD CHECK across macOS, Windows, and Ubuntu (multiple R versions) via GitHub Actions

## Important notes

- **NEVER run `devtools::document()`** — prompt the user to do it. Regenerating documentation requires the user's local environment.
- **Visual tests use vdiffr snapshots.** Updating snapshots requires interactive approval (`vdiffr::manage_cases()`). Do not attempt to update snapshot files directly.
- **Do not edit files in `man/`.** They are generated by roxygen2 from inline documentation in `R/` source files.
